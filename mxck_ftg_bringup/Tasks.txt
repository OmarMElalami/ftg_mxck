So benutzt du mxck_ftg_bringup
1) Vom PC auf den Jetson kopieren
scp -r .\mxck_ftg_bringup mxck@192.168.0.100:/home/mxck/mxck2_ws/src/
2) Im mxck2_control-Container bauen

Auf dem Jetson-Host:

sudo docker exec -it mxck2_control bash

Dann im Container:

source /opt/ros/humble/setup.bash
source /mxck2_ws/install/setup.bash
cd /mxck2_ws
colcon build --symlink-install --packages-select mxck_ftg_bringup
source /mxck2_ws/install/setup.bash
ros2 pkg executables mxck_ftg_bringup
3) Kompletten FTG-Stack mit einem Launch starten
sudo docker exec -it mxck2_control bash -lc "source /opt/ros/humble/setup.bash && source /mxck2_ws/install/setup.bash && ros2 launch mxck_ftg_bringup ftg_stack.launch.py"

4) Optional mit Front-Check-Debug starten, Wenn du zusätzlich scan_front_window_check sehen willst:
sudo docker exec -it mxck2_control bash -lc "source /opt/ros/humble/setup.bash && source /mxck2_ws/install/setup.bash && ros2 launch mxck_ftg_bringup ftg_stack.launch.py start_scan_check:=true"

5) Wenn TF schon woanders läuft

Dann kannst du TF im Bringup deaktivieren:

sudo docker exec -it mxck2_control bash -lc "source /opt/ros/humble/setup.bash && source /mxck2_ws/install/setup.bash && ros2 launch mxck_ftg_bringup ftg_stack.launch.py start_tf:=false"

6) Wenn mxck2_control die vehicle_control-Kette schon automatisch mitbringt:

sudo docker exec -it mxck2_control bash -lc "source /opt/ros/humble/setup.bash && source /mxck2_ws/install/setup.bash && ros2 launch mxck_ftg_bringup ftg_full_system.launch.py"

7) Wenn mxck2_control die vehicle_control-Kette schon automatisch mitbringt:

sudo docker exec -it mxck2_control bash -lc "source /opt/ros/humble/setup.bash && source /mxck2_ws/install/setup.bash && ros2 launch mxck_ftg_bringup ftg_full_system.launch.py"
 
8) Nur falls vehicle_control nicht schon läuft
sudo docker exec -it mxck2_control bash -lc "source /opt/ros/humble/setup.bash && source /mxck2_ws/install/setup.bash && ros2 launch mxck_ftg_bringup ftg_full_system.launch.py use_vehicle_control:=true"
