Richtiges Vorgehen

Auf dem Jetson Host:

sudo docker exec -it mxck2_control bash

Dann im Container:

source /opt/ros/humble/setup.bash
source /mxck2_ws/install/setup.bash
cd /mxck2_ws
colcon build --symlink-install --packages-select mxck_ftg_perception
source /mxck2_ws/install/setup.bash

mxck2_control ist dafür der richtige Ort, weil er laut Inspect/Compose auf dem echten mxck2_ws mit Humble läuft und /home/mxck/mxck2_ws:/mxck2_ws gemountet hat. Der control-Service startet außerdem nur bash, also ist er genau als manuelle Arbeitsumgebung geeignet.

Schritt F — prüfen, ob das Paket jetzt wirklich da ist

Noch im Container:

ros2 pkg executables mxck_ftg_perception
Erwartung:

Du solltest sehen:

mxck_ftg_perception scan_front_window_check
mxck_ftg_perception scan_preprocessor_node

Wenn das klappt, ist das Paket sauber installiert.

Schritt G — dann erst launchen

Terminal 2: TF laufen lassen wie bisher:

sudo docker exec -it mxck2_control bash -lc "source /opt/ros/humble/setup.bash && source /mxck2_ws/install/setup.bash && ros2 launch mxck_run broadcast_tf_launch.py"

In einem anderen Terminal:

sudo docker exec -it mxck2_control bash -lc "source /opt/ros/humble/setup.bash && source /mxck2_ws/install/setup.bash && ros2 launch mxck_ftg_perception scan_front_window_check.launch.py"

Dann:

sudo docker exec -it mxck2_control bash -lc "source /opt/ros/humble/setup.bash && source /mxck2_ws/install/setup.bash && ros2 launch mxck_ftg_perception scan_preprocessor.launch.py"
